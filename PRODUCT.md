# Product

## Register

product

## Users

Lyra Image Workbench serves creators who want a hosted image-generation workspace, end users who generate images with account credits, and administrators who configure upstream model access, free quotas, API keys, users, and operational settings.

## Product Purpose

The product turns image generation into a durable web workflow: users write or refine prompts, create background tasks, watch status and history, publish selected works, and reuse results. Success means a non-technical user can understand the shortest path to a generated image while operators can run the site safely as a small commercial image-generation service.

## Brand Personality

Clear, capable, direct. The interface should feel like a reliable creative tool rather than a marketing page.

## Anti-references

Avoid cramped dashboards, nested cards, decorative landing-page composition, hidden admin paths, ambiguous task states, distorted media previews, and node editors where the core workflow is squeezed into side panels.

## Design Principles

- Put the primary workflow in the largest visible area.
- Make task state, ownership, cost, and history explicit.
- Prefer familiar controls over invented affordances.
- Keep admin power visible only to administrators but easy for them to find.
- Use spacing and structure to reduce cognitive load before adding decoration.

## Accessibility & Inclusion

Target WCAG AA contrast for product UI text and controls. Support keyboard focus, reduced-motion users, responsive layouts, and color-independent status cues.
