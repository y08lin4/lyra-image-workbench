# Lyra Image Workbench 需求总账

> 本文用于跟踪当前 dev 分支的产品闭环需求、完成状态和验收口径。状态只代表当前实现视角，不代表最终验收通过。

## 已确认产品决策

| 决策 | 当前结论 | 后续影响 |
| --- | --- | --- |
| 主入口 | 创作画布是主入口 | 生成页降级为兼容/快捷入口，不能与画布抢主流程 |
| 视频功能 | 删除视频生成功能 | 2026-06-27 边界扫描：生产代码未见 MiniMax/video 入口、路由、组件、API、配置字段；历史文档残留只作为记录处理 |
| GIF/动图 | GIF/动图以后作为独立模块，不和视频混在一起 | 2026-06-27 边界扫描：当前无独立 GIF 工作台/API/FFmpeg 渲染链路；现有 GIF 仅作为图片 MIME/资产格式支持，未来若启用 FFmpeg 必须独立安全边界和版本提醒 |
| 作品保留 | 已提交广场作品不进 30 天清理；未提交广场生成结果需要真实后端清理 | 需要单节点后台清理任务，必须保守防误删 |
| 外部文档 | 站内 API 文档和外部 GitHub 文档仓库同步维护 | 站内示例、SDK 示例、给 AI 的提示词需同源口径 |
| 缺 Key 跳转 | 缺上游 Key 时跳转设置页是合理行为 | 设置入口保留，但重复设置窗口/重复后台入口要清理 |
| 生成页 vs 画布 | 二者暂时保留 | 创作画布是主流程，生成页作为兼容入口 |

## 验收与报告规则

1. 每个子代理完成后，必须回报改动文件、接口/页面、验证结果和风险。
2. 父线程负责整合，不能只转述子代理结果。
3. 合并前至少运行：
   - `go test ./...`
   - `npm run build -- --mode production`
   - `npx tsc --noEmit --pretty false --incremental false`
4. 如果浏览器自动化无法运行，需要明确说明，并给出手工验收页面列表。
5. 完成后更新本文状态，并追加到 `docs/current-round-task-report.md`。
