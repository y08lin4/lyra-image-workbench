# Security Policy

## 支持范围

当前仅维护 `master` 分支的最新版本。

## 报告安全问题

请不要通过公开 issue 披露安全漏洞、API Key、用户数据或可复现的攻击细节。

推荐流程：

1. 先通过私下渠道联系维护者，说明影响范围和最小复现步骤。
2. 等待确认和修复窗口。
3. 修复发布后再公开技术细节。

如果暂时没有私下联系方式，请创建一个不包含利用细节的 issue，标题写明“Security contact needed”。

## 本项目重点保护的信息

- 用户账号、2FA 密钥和会话 Cookie。
- 保存在 `data/` 下的空间配置、任务历史和云端 Key 状态。
- 上传参考图和 `outputs/` 生成结果。
- Debug 日志中的上游请求信息。
